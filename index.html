    Gulp

    A build can be triggered over your systems CLI like the launch command above. There are several tasks avaible (listed below), but if you want to re-build the whole application, simpy run the gulp command, and everything is done.

    To apply settings changed, you have to rebuild the application, simply rund the default gulp command to do this. 

    gulp # run whole build

# remove builded files, additionally use :app or
# :dependencies to just remove the related files
gulp clean:dist

# builds the app or dependencies only
gulp build:app
gulp build:dependencies

# watches the app or dependencies and auto-trigger
# the right builds on file changes
gulp watch:app
gulp watch:dependencies

    JSHint

    There's a gulp task integrated which helps you to lint the application files under /app. Simply run the task lint to validate all the files and generating an output.

    If there's a warning/info "ES5 option is now set per default" you can ignore this message and just keep on reading. This message will be removed in a newer version of JSHint and is generated due the esversion in the .jshintrc is set to 5 (using ECMAScript 5).

    Start

    The game can be started over your systems CLI, the only requirement is, that Node.js 4.+ is installed on your machine. Run the command below and see the magic. 

    npm start

    Phaser States

    States

    There are four general states avaible which are used for the game process. The Preload, Boot, Game and Update state. Read more about the states in the chapters below.

    Boot

    The boot state loads all image, sprites and audio assets in the preload function and sets the basic game settings in the create method. Define new media elements to get loaded within the world settings.

    Preload

    Used for pregame settings such as the game mode of phaser (Arcade in this case) and background settings ect. Visit the default settings here.

    Game

    Main game process, uses the game settings and the utility stuff. If you want to change something in the Jump N Run game itself, you'll have to change it here.

    Over

    This state will show the resume screen with the highscores. Afterwards it will run the Sync state.

    Sync

    At least the sync state will synchronize all the player scores to the file /app/data/scores.json via an AJAX request to HTTP POST /api/save/score.

    Emergency

    Sometimes you want to kill all players in the world or even quit the game itself due some unexpected bugs or lags. But you will loose the player scores if you just reload the browser window (press F5), so it is not recommended to reload the page to exit! There are two methods provided as soon as the game starts, they are located under the global Emergency object. Each method has to be confirmed with the yes button before it will affect your current game.

    $killAll

    This method will kill all alive players in your game, e.g. if one player lasts in the game and he hang up in an obstacle on the map and can't move further, this method will be great to kill him (or just use the one below). 

    Emergency.$killAll()

    $quit

    This method will use the $killAll method and additionally exits the game, saves the scores and display the overview like every player would die.

    You can also use the shortcut keycombination Ctrl + Y to quit the game! 

    Emergency.$quit()
